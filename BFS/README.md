# BFS
*Breadth First Search*<br><br>

**너비 우선 탐색**입니다. <br>
`implementation_1.cpp` 에서는 **격자 그래프 + 상하좌우 이동**에서의 BFS 예시 구현을<br>
`implementation_2.cpp` 에서는 **무향 / 유향 그래프 + 트리**에서의 BFS 예시 구현을 포함하고 있습니다.<br><br><br>

## 중요한 성질들
DFS와 달리 **Queue**를 사용해 탐색할 칸을 저장합니다.<br><br>

가중치가 없는 그래프일 시, **큐에서 먼저 꺼내지는 칸**이 **큐에서 나중에 꺼내지는 칸**보다 시작점에 **더 가까이 있습니다.** <br>
즉, 이를 사용하여 목적지까지의 **최단 거리**를 탐색할 때 사용 가능합니다.<br><br><br>

## 트리에서의 응용
임의의 점에서 **BFS**를 돌려서 나온 제일 먼 정점에서 다시 **DFS**를 돌려 나온 가장 먼 정점까지의 거리가 *트리의 지름*입니다.<br>
이는 트리에서의 가장 먼 두 점 사이의 거리를 의미합니다.<br><br>

Tree 상에서의 **Dynamic Programming**이 필요한 상태일 때,<br>
루트 노드에서 출발해 하위 노드를 탐색하는 과정을 거쳐야 한다면, **BFS**를 사용해야 하는 경우가 잦습니다.<br><br>

위에서의 **BFS**를 이용한 **Tree DP**일 때, 시작점을 정하는 문제이고 그 시작점을 가정했을 때의 **최솟값 / 최댓값** 과 같은 값을 구해야 하는 상황이라면<br>
**rerooting**이라는 테크닉을 이용해 `O(1)`만에 다른 시작점 상에서의 비용을 계산할 수 있습니다.<br><br><br>

## 그 외 느낀 점들
격자 그래프 상에서의 상하좌우를 탐색할 때, `direction_x[4]` 와 `direction[4]` 배열을 만들어 상하좌우 탐색을 편리하게 하는 방식이 효율적인 것 같습니다.<br>
이는 상하좌우 이동에서만 사용되는 것이 아니라, **대각선 이동을 포함한 8방향 이동**, **체스 기물중 하나인 나이트의 이동** 등을 구현할 때에도 비슷한 구현을 사용해 코드를 간결하게 만들 수 있습니다.<br><br>

탐색을 하며 **Queue**를 채워나갈 때, 조건을 만족하는 칸을 `push`하는 것이 아니라 `push`한 후 조건을 검사하는 구현을 했을 때가 종종 있었습니다.<br>
이는 탐색의 효율을 저하시키고, 디버깅을 할 때 불편함을 초래할 수 있으니 주의하시는 것이 좋습니다.<br><br>

`implementation_2.cpp`의 경우와 같이 **인덱스는 필요 없고 순회할 원소만 필요한 경우**, <br>
인덱스 기반의 반복문 대신 *iterator를 사용한 range based loop* 사용이 편리한 것 같습니다.<br>
위의 문법은 **C++ 11 이상**의 버전을 요구합니다.
